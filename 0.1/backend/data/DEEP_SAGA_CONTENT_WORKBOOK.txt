DEEP SAGA CONTENT WORKBOOK
==========================

Purpose
-------
Use this file to describe your game content without writing SQL or code.
Fill in the blank templates and send the completed file back for import.

The database is the source of truth. The AI may narrate this content, but it
must not invent permanent monsters, NPCs, bosses, items, skills, or locations.


WORLD CONTENT TARGET
--------------------

10 Dungeons / Realms
  -> 5 Floors in each realm
  -> 50 Floors total
  -> 1 authored Floor Story per floor
  -> 50 Floor Stories total
  -> About 2 or more NPCs per floor
  -> 100 or more NPCs total
  -> About 6 or more monster types per floor
  -> 300 or more Monsters total
  -> Floor 5 contains the realm's Main Boss
  -> 10 Main Bosses total
  -> About 10 Items per floor
  -> 500 Items total
  -> 200 Skills total

A "Dungeon" does not need to be an underground dungeon. It can be a kingdom,
forest, academy, ocean, city, spirit world, battlefield, library, or any other
place. In this workbook, "Realm" means the same thing as "Dungeon".

FLOOR PURPOSE RULES
-------------------

Each realm normally follows this rhythm:

Floor 1: Arrival and introduction
Floor 2: Puzzle, mystery, lore, or exploration
Floor 3: NPC conflict, quest, or moral decision
Floor 4: Dangerous challenge, chase, war, or mini-boss
Floor 5: Main Boss and realm conclusion

You can change the purpose when needed, but Floor 5 must remain the Main Boss
floor. Include quiet scenes such as campfires, libraries, meals, travel, and
personal conversations so every floor is not only combat.

SKILL UNLOCK RULES
------------------

Yes: most skills should be earned from player actions, behavior, training,
discoveries, quests, race/class traits, bosses, items, or soul memories.

The AI must not award a skill because it sounds interesting. The intended flow:

1. Player attempts an action.
2. The engine checks the current scene and character abilities.
3. The action succeeds, partially succeeds, or fails.
4. The engine records the action and result.
5. The engine compares that history with skill requirements in the database.
6. If every requirement is met, the engine adds the skill to character_skills.
7. The AI narrates the discovery using the saved skill description.

Examples:

- Climb during danger several times -> unlock movement or climbing skill.
- Repeatedly protect companions -> unlock Guardian Instinct passive.
- Negotiate with monsters -> unlock Monster Tongue or Empathic Pressure.
- Win fights without attacking first -> unlock Counter Stance.
- Study an enemy before every fight -> unlock Hunter's Analysis.
- Use poison carelessly -> possibly gain poison resistance and a curse.
- Train with a named NPC -> unlock only the skills that NPC knows.
- Defeat a boss under a special condition -> unlock that boss reward skill.
- Remember danger across reincarnations -> unlock soul-memory skills.

Creative typed actions should count when they are possible in the current state.
Repeated meaningless text, impossible actions, or failed actions should not count.

IMPORTANT CURRENT IMPLEMENTATION NOTE
-------------------------------------

The database already stores:

- Skill definitions and requirements
- Character-owned skills
- Skill level and number of uses
- Player choices and parsed intent
- Story memories and consequences

New characters currently receive the starter skills Brace and Soul Echo.
The AI can currently suggest newItemsOrSkills in its response, but the backend
does not yet automatically validate those suggestions against requirements and
insert the skill. The action-based skill evaluator is the next engine module
needed. Until it is built, action-based skill unlocking is the intended design,
not a fully enforced feature.
